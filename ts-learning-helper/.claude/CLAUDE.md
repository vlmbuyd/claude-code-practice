# .claude/CLAUDE.md

Claude Code 에이전트를 위한 지침입니다.

## 프로젝트 목적

이펙티브 타입스크립트를 읽으며 생기는 **궁금증에 답변하고**, 개념 이해를 위해 예제 코드를 탐구하도록 돕는 CLI입니다.

"문제 풀이 도구"가 아닙니다. 사용자가 책을 읽다 막히거나 궁금한 개념이 생겼을 때 질문하고, 필요하면 관련 코드 예제를 직접 써보며 이해를 심화하는 것이 핵심 흐름입니다.

## 시스템 프롬프트 수정 시 확인사항

`src/prompts/` 파일들이 제품의 핵심 가치입니다. 수정 시 다음을 확인하세요:

1. 사용자의 학습 전략 8가지에 부합하는가?
2. 사용자의 약점을 보완하는 행동 제약이 포함되어 있는가?
   - 한 번에 너무 많은 것을 이해하려는 욕심 → 키워드 수 제한, 개선점 1개만
   - 메타인지 부족 → 자기 설명 요청 (파인만 기법의 R 단계)
   - 단계 분해 어려움 → 명확한 단계 구조 강제
3. 초점이 충분한가? 포괄적 나열이 아닌 핵심에 집중하는가?

## 커맨드 추가 패턴

항상 이 순서를 따르세요:
1. 프롬프트 파일 (`src/prompts/`)
2. 커맨드 파일 (`src/commands/`)
3. `src/index.ts`에 등록
4. `CLAUDE.md`에 문서화

## 사용자 학습 전략 요약

1. 책을 읽으며 생기는 궁금증을 질문 → `explain` 커맨드
2. 챕터/아이템의 핵심 키워드 파악 → `keywords` 커맨드
3. 개념을 코드로 직접 확인 (스스로 먼저 작성 → AI 보조) → `solve` 커맨드
4. 작성한 코드 피드백 → `feedback` 커맨드
5. Feynman Technique으로 개념 체화 → `explain --feynman`
6. 대화 중 생긴 인사이트 기록 → `insight` 커맨드
7. 챕터 학습 진도 관리 → `progress` 커맨드

## solve 커맨드 사용 의도

`solve`는 "모르는 문제를 AI에게 풀어달라는" 커맨드가 아닙니다.
개념을 배운 후 **"이 개념을 코드로 어떻게 표현하는가?"** 를 직접 시도해보고,
작성한 코드에 대해 AI의 피드백을 받는 탐구 도구입니다.
