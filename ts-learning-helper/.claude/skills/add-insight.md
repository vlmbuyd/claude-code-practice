# Add Insight Skill

대화 중 발생한 "아하!" 순간을 즉시 기록합니다.

## 사용법

```
/add-insight
```

## 진행 순서

1. **대화 분석**: 최근 대화에서 가장 중요한 인사이트 후보 1-2개를 한 문장으로 요약하세요.

2. **확인 요청**: "이 인사이트를 저장할까요?" 라고 제안하세요:
   - 제안 1: [추출한 인사이트 문장]
   - 또는 직접 입력

3. **저장 실행**: 확정된 인사이트로 `npm run dev -- insight add "<인사이트>"` 를 실행하세요. 대화에서 챕터가 파악됐다면 `--chapter <n>` 옵션을 추가하세요.

## 좋은 인사이트의 기준

- 한 문장으로 완결됨
- 구체적인 TypeScript 개념과 연결됨
- 나중에 읽었을 때 "왜 이게 중요한지" 바로 이해됨
- "X는 Y이다" 또는 "X를 사용하면 Z를 피할 수 있다" 형태

## 예시

좋은 인사이트:
- "타입 가드는 런타임 검사를 컴파일 타임으로 연결하는 다리 역할을 한다"
- "`as`보다 타입 가드를 쓰면 런타임에서도 안전성이 보장된다"

나쁜 인사이트:
- "타입스크립트는 어렵다" (너무 추상적)
- "Item 22를 공부했다" (내용 없음)
